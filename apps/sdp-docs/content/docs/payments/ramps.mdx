---
title: Ramps
description: Fiat on-ramps and off-ramps — let users buy crypto with fiat or cash out from a wallet through SDP.
---

SDP wraps several ramp providers behind a two-step flow: **estimate** a rate without committing to anything, then **quote** to create a real, provider-backed order. There is no single `execute` call — on-ramp and off-ramp each have their own estimate and quote endpoints:

| Step | On-ramp | Off-ramp |
| --- | --- | --- |
| Estimate | `POST /v1/payments/ramps/onramp/estimate` | `POST /v1/payments/ramps/offramp/estimate` |
| Quote | `POST /v1/payments/ramps/onramp/quote` | `POST /v1/payments/ramps/offramp/quote` |

Estimate is unauthenticated against any provider and fans out across every provider that supports the requested pair — no counterparty required. Quote is provider-specific: you pick one `provider` and it returns either a hosted redirect URL, funding instructions to display, or a session-widget secret, depending on `deliveryMode`.

For per-provider configuration and which providers support which direction, see [Ramp providers](/docs/payments/ramps-providers).

## Prerequisites

Quoting requires an existing counterparty (`POST /v1/counterparties`) representing the end-user. Provider-specific KYC/compliance fields are collected just-in-time via `POST /v1/counterparties/{counterpartyId}/requirements` before the counterparty is "ready" for a given provider and direction — if you quote before that, the call fails with a `409 CONFLICT` (`counterpartyNotProvisioned`). Estimate does not need a counterparty.

## Estimate

Estimate takes a Solana crypto rail and a fiat currency and returns a per-provider rate preview. It never touches a counterparty and never creates a transfer record.

<Tabs items={["curl", "TypeScript"]}>
<Tab value="curl">
```bash
curl -X POST https://sdp-api-production.solana.workers.dev/v1/payments/ramps/onramp/estimate \
  -H "Authorization: Bearer sk_test_..." \
  -H "Content-Type: application/json" \
  -d '{
    "assetRail": "usdc.solana",
    "fiatCurrency": "USD",
    "fiatAmount": "100.00"
  }'
```
</Tab>
<Tab value="TypeScript">
```typescript
const response = await fetch(
  "https://sdp-api-production.solana.workers.dev/v1/payments/ramps/onramp/estimate",
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer sk_test_...",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      assetRail: "usdc.solana",
      fiatCurrency: "USD",
      fiatAmount: "100.00",
    }),
  }
);
const { data } = await response.json();
// data.estimates: one entry per provider that supports USD -> usdc.solana
```
</Tab>
</Tabs>

Off-ramp estimate is the mirror image — `assetRail` + `fiatCurrency` + `cryptoAmount` on `POST /v1/payments/ramps/offramp/estimate`.

### Estimate request fields

| Field | Notes |
| --- | --- |
| `assetRail` | Solana crypto rail: one of `sol.solana`, `usdc.solana`, `usdt.solana`, `usdg.solana`, `pyusd.solana`. |
| `fiatCurrency` | ISO 4217 fiat currency code. Validated against SDP's ramp fiat-currency list, uppercased automatically — actual availability for a given rail is per-provider (see [Ramp providers](/docs/payments/ramps-providers)). |
| `fiatAmount` (on-ramp) / `cryptoAmount` (off-ramp) | Decimal string, e.g. `"100.00"`. |

### Estimate response

```json
{
  "estimates": [
    { "provider": "moonpay", "status": "ok", "estimate": { "provider": "moonpay", "direction": "onramp", "fiatCurrency": "USD", "assetRail": "usdc.solana", "fiatAmount": "100.00", "cryptoAmount": "98.50", "exchangeRate": "0.985", "fees": { "currency": "USD", "total": "1.50" } } },
    { "provider": "lightspark", "status": "unsupported" },
    { "provider": "bvnk", "status": "error", "error": "Provider timed out" }
  ]
}
```

Each entry in `estimates` is one of three shapes, discriminated by `status`:

- **`ok`** — the provider returned a live rate. `estimate.cryptoAmount` / `estimate.fiatAmount` are the converted amounts, `estimate.fees` describes the fee currency and total, and `estimate.minFiatAmount` / `estimate.maxFiatAmount` (when present) are the provider's amount limits.
- **`unsupported`** — the provider supports this pair but only reports a rate at quote time (no estimate preview available).
- **`error`** — the provider call failed; `error` is a human-readable message.

Providers that don't support the pair at all are omitted from `estimates` entirely — check `GET /v1/payments/ramps/onramp/currency` (or `.../offramp/currency`) to discover supported pairs up front instead of guessing.

## Quote

Quote commits to one provider and creates a real order. Unlike estimate, `cryptoToken` here is a currency **symbol** (`USDC`), not a rail id (`usdc.solana`) — the two endpoints use different vocabularies for the same asset.

<Tabs items={["curl", "TypeScript"]}>
<Tab value="curl">
```bash
curl -X POST https://sdp-api-production.solana.workers.dev/v1/payments/ramps/onramp/quote \
  -H "Authorization: Bearer sk_test_..." \
  -H "Content-Type: application/json" \
  -d '{
    "provider": "moonpay",
    "counterpartyId": "counterparty_example",
    "destinationWallet": "privy_wallet_123",
    "cryptoToken": "USDC",
    "fiatCurrency": "USD",
    "fiatAmount": "100.00",
    "redirectUrl": "https://app.example.com/onramp/complete"
  }'
```
</Tab>
<Tab value="TypeScript">
```typescript
const response = await fetch(
  "https://sdp-api-production.solana.workers.dev/v1/payments/ramps/onramp/quote",
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer sk_test_...",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      provider: "moonpay",
      counterpartyId: "counterparty_example",
      destinationWallet: "privy_wallet_123",
      cryptoToken: "USDC",
      fiatCurrency: "USD",
      fiatAmount: "100.00",
      redirectUrl: "https://app.example.com/onramp/complete",

```

Off-ramp mirrors it on `POST /v1/payments/ramps/offramp/quote`, swapping `destinationWallet` for `sourceWallet` (the wallet you're cashing out from) and `fiatAmount` for `cryptoAmount`:

```json
{
  "provider": "bvnk",
  "counterpartyId": "counterparty_example",
  "sourceWallet": "privy_wallet_123",
  "cryptoToken": "USDC",
  "fiatCurrency": "USD",
  "cryptoAmount": "100.00"
}
```

### Quote request fields

| Field | Required | Notes |
| --- | --- | --- |
| `provider` | yes | One of `moonpay`, `lightspark`, `bvnk`, `moneygram`, `coinbase`, `mural`, `stripe`. Not every provider supports both directions — see [Ramp providers](/docs/payments/ramps-providers#capability-matrix). |
| `counterpartyId` | yes | An existing SDP counterparty. Provider-native customer records are resolved or created from it. |
| `destinationWallet` (on-ramp) / `sourceWallet` (off-ramp) | yes | A `walletId` from `GET /v1/wallets` (for example `privy_wallet_123`) **or** a Solana address. |
| `cryptoToken` | yes | Crypto currency symbol, e.g. `USDC`. |
| `fiatCurrency` | on-ramp: yes; off-ramp: provider-dependent | ISO 4217 fiat currency code. |
| `fiatAmount` (on-ramp) / `cryptoAmount` (off-ramp) | yes | Decimal string. |
| `redirectUrl` | no | Return URL after a hosted provider flow completes. |
| `domain` | no | On-ramp only. Browser origin host, used for Coinbase's Apple Pay payment link. |

### Quote response

```json
{
  "quote": {
    "id": "ramp_quote_example",
    "provider": "moonpay",
    "status": "pending",
    "deliveryMode": "hosted",
    "hostedUrl": "https://buy.moonpay.com/...",
    "exchangeRate": 0.985,
    "totalSendingAmount": 10000,
    "sendingCurrency": { "code": "USD", "decimals": 2 },
    "totalReceivingAmount": 9850,
    "receivingCurrency": { "code": "USDC", "decimals": 6 },
    "expiresAt": "2026-07-18T12:00:00.000Z"
  }
}
```

`deliveryMode` tells you what to do with the quote:

- **`hosted`** — open `hostedUrl`; the provider runs KYC and payment-method capture, then settles crypto to the destination wallet (on-ramp) or pays out fiat (off-ramp).
- **`manual_instructions`** — display `paymentInstructions` (bank details or a crypto deposit address, shape varies by provider) and wait for the user to fund it.
- **`session_widget`** — mount the provider's SDK using `clientSecret` / `sessionId` / `publishableKey` (Stripe on-ramp).

`status` is one of `pending`, `processing`, `completed`, `failed`, reflecting the provider's state at the moment the quote call returns. SDP does not poll the provider afterward — observe settlement via the resulting transfer (see below) or a pending quote's expiry (`expiresAt`).

## Canceling a pending quote

`POST /v1/payments/ramps/transfers/cancel` cancels a quote-backed transfer that is still `pending` or `awaiting_payment`, identified by `provider` + `providerReference`:

```json
{ "provider": "bvnk", "providerReference": "Quote:019e979c-f660-5246-0000-c0588496b9ce" }
```

## Observing settlement

A successful quote's on-chain leg surfaces as a payment transfer. List ramp transfers with `GET /v1/payments/transfers?category=ramp` (add `provider` + `providerReference` to look up one quote's transfer exactly), or find the resulting inbound transfer on the destination custody wallet via [Accept payments](/docs/payments/accept-overview). SDP does not currently emit settlement webhooks to your application for ramp transfers.

## Provider selection

You always pass `provider` explicitly; SDP does not auto-select. Choose based on:

- **Coverage** — call `GET /v1/payments/ramps/onramp/currency` / `.../offramp/currency` to see which providers support a given fiat/rail pair for your organization right now, or see the static roster in [Ramp providers](/docs/payments/ramps-providers).
- **Direction** — not every provider supports both on-ramp and off-ramp (see the [capability matrix](/docs/payments/ramps-providers#capability-matrix)).
- **Sandbox needs** — sandbox vs production is chosen per request from the calling API key's environment (`sk_test_` vs `sk_live_`); see [Sandbox vs production](/docs/payments/ramps-providers#sandbox-vs-production-selection).
